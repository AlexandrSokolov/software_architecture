### As application user bases grew, how did the underlying approach to persisting their data fundamentally change?
<details><summary>Show answer</summary>

With web and mobile apps, software as a service (SaaS), and cloud services,
it has become normal to store data from many different users **in a shared server-based data infrastructure**.

</details>

### Why does a single machine eventually stop being a viable home for an application's data?
<details><summary>Show answer</summary>

Two pressures force the move off a single machine:

- **Data volume grows** — it no longer fits on one machine.
- **Query rate grows** — one machine can't serve the load.

The only remedy is **distribution across multiple machines**.

</details>
